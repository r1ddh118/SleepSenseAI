# SleepSense AI — Regulatory Pathway Notes

## Current Positioning: Wellness / Screening Tool

No regulatory approval required for a screening/wellness claim in many jurisdictions. This allows commercial launch while collecting real-world evidence. **Confirm with counsel** for each market.

## India — CDSCO SaMD Pathway

- Regulator: Central Drugs Standard Control Organisation
- Classification: Class B or C Software as a Medical Device (SaMD)
- Relevant rule: Medical Devices Rules 2017 (amended 2020)
- Required: Technical file, clinical evidence, QMS (ISO 13485)
- Timeline: often 12–18 months after IRB-quality data, depending on class
- Start with: Manufacturer registration + SaMD classification request

## USA — FDA Pathway

- Likely pathway: De Novo (novel SaMD without predicate) or 510(k)
- Possible predicate for 510(k) discussion: peripheral devices used for OSA screening (e.g. WatchPAT-class arguments) — **predicate choice requires regulatory counsel**
- Required: Clinical/performance data, software lifecycle documentation (e.g. IEC 62304-aligned)
- Timeline: highly variable (often 18–24+ months for first clearance)

## EU — MDR CE Marking

- Classification: commonly IIa for non-invasive diagnostic support (not legal advice)
- Requires: Notified Body (as applicable), Clinical Evaluation Report, UDI registration
- Timeline: often 18–24+ months depending on class and NB queue

## Recommended Sequence (illustrative)

1. Launch as wellness / screening tool → collect real-world data under appropriate terms
2. Complete IRB prospective study (Phase 4)
3. Publish clinical evidence (peer review)
4. File CDSCO SaMD (India) when dossier-ready
5. File FDA De Novo or 510(k) when US strategy is fixed
